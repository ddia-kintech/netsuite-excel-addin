NetSuite Excel Add-in - Support

Welcome to the support page for the NetSuite Excel Add-in. Here you'll find answers to common questions and troubleshooting tips.

Common Issues

The add-in doesn't appear in Excel after installation

Try these troubleshooting steps:

  1. Make sure you're using a supported version of Excel (Excel 2016 or later)
  2. Check if the manifest XML file was properly uploaded
  3. Restart Excel completely
  4. Check if the add-in is listed under "My Add-ins" in Excel
I can't connect to my NetSuite account

Verify the following:

  1. Your NetSuite Account ID is correct (including any _SB1 suffix for sandbox)
  2. Your Token-Based Authentication credentials are valid
  3. The Integration record in NetSuite has the correct permissions
  4. Your role in NetSuite has appropriate permissions
Custom forms aren't appearing in the dropdown

This could be due to:

  1. The selected subsidiary doesn't have custom forms assigned
  2. Your NetSuite role doesn't have permission to access these forms
  3. The forms exist but aren't available for the selected transaction type

NetSuite Setup Instructions

To use this add-in, you'll need to set up Token-Based Authentication (TBA) in NetSuite:

  1. Log in to NetSuite as an administrator
  2. Go to Setup > Integration > Manage Integrations
  3. Click "New" to create a new integration record
  4. Enter a name (e.g., "Excel Add-in")
  5. Check "Token-Based Authentication"
  6. Save the record and note the Consumer Key/Secret
  7. Go to Setup > Users/Roles > Access Tokens
  8. Click "New" to create a new token
  9. Select your user and the integration you created
  10. Save the record and note the Token ID/Secret
Note: The Consumer Key/Secret and Token ID/Secret should never be shared. They provide access to your NetSuite account.
